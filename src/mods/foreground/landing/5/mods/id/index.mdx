## Apa ini?

<div className="h-2"/>

Kami memastikan bahwa kami selalu memiliki keadaan blockchain yang sebenarnya dengan menggunakan algoritma yang menciptakan konsensus dari beberapa server. Ini berarti bahwa jika satu server dikompromikan dan berbohong tentang keadaan blockchain, kami dapat mendeteksinya dengan membandingkan keadaannya dengan keadaan server lain. Ini mirip dengan cara blockchain itu sendiri bekerja, tetapi dengan jumlah server yang lebih sedikit.

<div className="h-4"/>

## Bagaimana cara kerjanya?

<div className="h-2"/>

Kami menggunakan server publik dari berbagai penyedia dan lokasi yang berbeda untuk mencegah titik kegagalan tunggal. Setiap permintaan dikirim ke beberapa server, dan hasilnya dibandingkan untuk memastikan bahwa responsnya benar. Jika ada ketidaksesuaian, kami lebih memilih respons dari mayoritas server. Jika mayoritas tidak ditemukan, kami lebih memilih untuk tidak menggunakan respons sama sekali.
